## Analysis
The user wants to modify `cl_eyes.lua` to change the visual feedback when both eyes are injured (bilateral blindness). 

Currently, the code in `hg.organism.DrawEyeBlindOverlay` attempts to draw a single black rectangle covering the whole screen when both eyes are blind, but it uses a clamped `intensity` value for the alpha channel, making it partially transparent (between 200 and 252). The user specifically requested that this overlay be **not transparent** (i.e., fully opaque, alpha 255).

## Technical Feasibility
High. This is a simple modification of a single line in a client-side Lua file.

## Implementation Approach
1. **Target File**: `lua/homigrad/organism/tier_1/cl_eyes.lua`
2. **Target Function**: `hg.organism.DrawEyeBlindOverlay`
3. **Modification**: In the block `if blindL and blindR then`, change `surface.SetDrawColor(0, 0, 0, math.Clamp(math.Round(intensity * 255), 200, 252))` to `surface.SetDrawColor(0, 0, 0, 255)`.

## Edge Cases & Potential Issues
- **Existing functionality**: The `drawScreenEyeBlind` function is still used for single-eye blindness, which remains partially transparent. This is correct according to the user's request, as they only specified the "both eyes are injured" case.
- **Visual impact**: A fully black screen might be jarring, but it's exactly what was requested ("not transparent").

## Testing Strategy
Since this is a client-side visual effect in a GMod environment, automated unit testing is not feasible. Verification will rely on:
- Manual verification by an agent/user (if possible) or simply ensuring the code is syntactically correct and meets the logic requirements.
